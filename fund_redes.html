Fundamentos de Redes

Sobre a matéria de Fundamentos de Redes

Os Fundamentos de Redes estudam como dispositivos se conectam e trocam informações, formando sistemas de comunicação essenciais no mundo digital. Esse conteúdo aborda desde o que são redes e internet até os componentes físicos e lógicos envolvidos. Também inclui modelos e protocolos que organizam o tráfego de dados, como TCP/IP e HTTP. Com isso, é possível entender como computadores, servidores e aplicações se comunicam de forma eficiente e segura.

O que são Redes de Computadores

Redes de computadores são conjuntos de dispositivos interligados (computadores, servidores, roteadores) que compartilham dados e recursos. Elas permitem comunicação, transferência de arquivos e acesso a serviços remotos. Com redes, vários usuários podem usar simultaneamente internet, impressoras ou bancos de dados. A estrutura da rede define velocidade, alcance e confiabilidade da comunicação.

O que é uma Rede mínima

Uma rede mínima é o arranjo mais simples possível de computadores interligados, como por exemplo dois dispositivos conectados diretamente. Serve como base para estudar conceitos como comunicação, endereçamento e troca de dados. Mesmo simples, já permite troca de arquivos e mensagens entre máquinas. É útil para entender os fundamentos antes de redes maiores e mais complexas.

O que é a internet o que compõe

A internet é uma rede mundial de redes, um sistema interligado de milhões de computadores, servidores e redes em todo o planeta. Ela é composta por backbones, cabos submarinos e terrestres, provedores, data centers, roteadores e protocolos de comunicação. A internet permite acesso global a informações, serviços e comunicação em tempo real entre dispositivos.

O que é web

A Web (World Wide Web) é o conjunto de páginas, sites e aplicações acessíveis pela internet usando navegadores. É apenas uma das muitas formas de uso da internet. A Web usa protocolos e recursos como HTTP/HTTPS, página HTML, links, servidores e clientes para exibir conteúdos, vídeos, textos e serviços online

O que é o mapa de fibra óptica

O mapa de fibra óptica global mostra os cabos submarinos e terrestres que transportam dados entre continentes e países. a “espinha dorsal” da internet mundial.


Esses cabos conectam servidores e provedores pelo mundo, permitindo que dados circulem globalmente com alta velocidade e baixa latência.

Quais os componentes de hardware de uma rede wi-fi

Uma rede Wi-Fi depende de hardware como roteador ou ponto de acesso sem fio, antenas, placa de rede sem fio nos dispositivos (notebook, celular, etc.) e, às vezes, repetidores ou extensores para ampliar o alcance. Também pode haver modems para conectar à internet via provedor. Esses componentes fazem com que os dispositivos comuniquem-se entre si e acessem a internet sem cabos.

Quais os componentes de hardware de uma rede local com cabo de rede

Em uma rede local cabeada, são usados cabos de rede (como cabo Ethernet), switches ou hubs para conectar vários dispositivos, placas de rede nas máquinas, e roteador ou modem para ligação externa. Esse tipo de rede costuma ser mais estável e rápida do que a sem fio. As placas, cabos e equipamentos garantem comunicação confiável e compartilhamento de recursos.

O que é um protocolo de comunicação

Protocolo de comunicação é um conjunto de regras e padrões que definem como os dispositivos trocam dados em formato de pacotes, sinais, endereços, erros e confirmações. Protocolos garantem que remetente e destinatário entendam a informação, mesmo utilizando hardware e sistemas diferentes. Sem protocolos, a comunicação entre redes e máquinas seria caótica ou impossível.

O que é o modelo OSI

O modelo OSI (Open Systems Interconnection) é uma referência teórica que divide a comunicação em 7 camadas, desde a parte física até a aplicação. Cada camada tem responsabilidades específicas como transmissão física, roteamento, controle de dados, sessão e aplicação. Ele ajuda a entender, planejar e diagnosticar redes, definindo como dados devem viajar e ser processados.

E o modelo TCP/IP

O modelo TCP/IP é uma arquitetura prática e usada de fato na internet. Ele simplifica o modelo OSI em 4 camadas (link, rede, transporte e aplicação) e define protocolos como TCP, IP, HTTP etc. É a base de comunicação da internet moderna, governando como pacotes viajam entre dispositivos, garantindo entrega e integridade dos dados.

O que é o protocolo IPV4 e IPV6

IPv4 (Internet Protocol version 4) usa endereços de 32 bits, limitados (~4,3 bilhões de endereços), comuns em quase todas redes atualmente. Permite identificação e roteamento de dispositivos na internet.
IPv6 é a versão mais nova, com endereços de 128 bits, oferecendo uma quantidade praticamente ilimitada de endereços. tornou-se essencial com o crescimento da internet e de dispositivos conectados.

Como é configurado o IP e o TCP em uma programação backend com node.js

No backend em Node.js, o IP e o TCP são configurados principalmente através do módulo net ou de frameworks como Express, que usam o protocolo TCP internamente. O servidor é criado escutando um endereço IP e uma porta específica, definidos no código (por exemplo, server.listen(port, host)). O Node gerencia a comunicação TCP automaticamente, permitindo enviar e receber dados por sockets. Assim, o desenvolvedor apenas define o IP, a porta e as regras de tratamento das conexões.

O que é o protocolo HTTP e HTTPS

O protocolo HTTP é o padrão de comunicação utilizado na web para enviar e receber dados entre cliente e servidor, porém sem criptografia. Já o HTTPS é a versão segura do HTTP, pois utiliza SSL/TLS para criptografar as informações transmitidas. Isso garante autenticidade, privacidade e proteção contra interceptações. Ambos são fundamentais para o funcionamento dos sites e serviços online.

Exemplo de um framework para programação do protocolo HTTP com uso do node.js

Um dos frameworks mais usados para programar o protocolo HTTP em Node.js é o Express. Ele simplifica a criação de servidores, permitindo definir rotas, lidar com requisições e respostas e integrar facilmente middleware. Com o Express, é possível construir APIs e aplicações web de forma rápida e organizada. É leve, flexível e amplamente adotado na comunidade.

    const email_bd = 'carlos@gmail.com'
    const senha_bd = 123

    const dados = {
        email: email_bd,
        senha: senha_bd
    }

    res.status(200).json(dados)

app.post('/login', (req,res)=>{
    const email_bd = 'carlos@gmail.com'
    const senha_bd = 123
    const valores = req.body

    console.log(valores)

    if(email_bd === valores.email){
        if(senha_bd === valores.senha){
            return res.status(200).json({ 
                message: 'login realizado com sucesso!', 
                nome: valores.nome, 
                statusLog: 'true'
            })
        }else{
            return res.status(403).json({ 
                message: 'Acesso não autorizado!'
            })
        }
    }else{
        return res.status(404).json({
            message: 'Usuário não encontrado'
        })
    }
})
